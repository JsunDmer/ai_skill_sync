# Skill 同步工具 - 需求文档

**创建日期**: 2026-04-10  
**状态**: 已完成头脑风暴  
**版本**: 1.0

---

## 1. 概述

创建一个技能同步工具，实现技能的中央存储、全平台共享和可视化管理的全栈 Web 应用。

### 核心价值主张
- 一份技能定义，全平台共享（OpenCode、Claude、Trace-CN、Cursor）
- Git 仓库作为中央存储backend
- 前端可视化界面进行技能管理

### 目标用户
- 需要在多个 AI Agent 平台中使用相同技能的用户
- 需要统一管理技能的用户
- AI Agent 深度用户

## 2. 功能需求

### 2.1 技能管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 查看技能列表 | 在前端展示所有已同步的技能 | P0 |
| 查看技能详情 | 点击查看技能的完整内容（名称、描述、文件路径、代码内容） | P0 |
| 添加技能 | 将本地或远程的技能添加到中央仓库 | P0 |
| 删除技能 | 从中央仓库删除指定技能 | P0 |
| 编辑技能元数据 | 修改技能的名称、描述、标签 | P1 |
| 技能搜索 | 通过关键词搜索技能 | P1 |

### 2.2 Git 同步

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 链接 Git 仓库 | 配置中央技能仓库地址 | P0 |
| 拉取最新 | 从 Git 仓库拉取最新技能 | P0 |
| 提交更改 | 将本地修改提交到 Git 仓库 | P0 |
| 冲突处理 | 处理多端修改带来的冲突 | P1 |

### 2.3 平台适配

| 功能 | 描述 | 优先级 |
|------|------|--------|
| OpenCode 适配器 | 技能格式转换为 OpenCode 可用格式 | P0 |
| Claude Code 适配器 | 技能格式转换为 Claude Code 可用格式 | P0 |
| Trace-CN 适配器 | 技能格式转换为 Trace-CN 可用格式 | P0 |
| Cursor 适配器 | 技能格式转换为 Cursor 可用格式 | P0 |

### 2.4 前端功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 技能列表页 | 展示所有技能卡片 | P0 |
| 技能详情页 | 展示单个技能的完整内容 | P0 |
| 添加技能页面 | 表单添加新技能 | P0 |
| 设置页面 | Git 仓库配置、平台选择 | P0 |
| 平台预览 | 预览技能在各平台的格式 | P1 |

---

## 3. 非功能需求

### 3.1 性能
- 页面加载时间 < 2s
- 技能同步操作 < 5s

### 3.2 兼容性
- 现代化浏览器（Chrome、Firefox、Safari、Edge 最新版）
- 移动端响应式布局

### 3.3 数据格式

技能文件格式（统一格式）：
```json
{
  "name": "skill-name",
  "description": "技能描述",
  "platforms": ["opencode", "claude", "trace-cn", "cursor"],
  "content": {
    "filePath": "相对路径",
    "content": "技能内容"
  },
  "tags": ["tag1", "tag2"],
  "createdAt": "2026-04-10",
  "updatedAt": "2026-04-10"
}
```

## 4. 技术选择

### 推荐技术栈
- **前端框架**: Next.js 14 (App Router)
- **UI 库**: Tailwind CSS + shadcn/ui
- **状态管理**: React Context / Zustand
- **Git 操作**: isomorphic-git 或 simple-git
- **部署**: Vercel 或自托管

### 项目结构
```
ai_skill_sync/
├── src/
│   ├── app/              # Next.js App Router
│   ├── components/         # React 组件
│   ├── lib/               # 工具函数
│   ├── hooks/             # 自定义 hooks
│   ├── stores/            # 状态管理
│   └── styles/            # 全局样式
├── public/                # 静态资源
├── docs/                  # 文档
└── tests/                 # 测试
```

## 6. 已知风险

1. **Git 冲突**: 多端同时修改可能导致冲突，需要考虑冲突处理机制
2. **平台差异**: 不同平台对技能格式的支持程度不同，适配器需要处理这些差异
3. **认证**: Git 仓库的认证机制（SSH/HTTPS）需要支持
